0 · WHY

你的 AI 助手为什么总是绑在工位？

每个用过 agent 类 AI 编程工具的人都遇到过：晚上躺床上想到一句改的话，要不要起来开电脑？地铁上想继续白天的活，看着手机干瞪眼。

问题不在 AI，在你触达 AI 的方式

主流 AI 编程 agent（CodeBuddy / Cursor / Claude Code）都跑在桌面 IDE 里。它有三个特点决定了"必须在电脑前用"：

  • 上下文锁在内存里：会话一关上下文丢光，下次还得从头讲
  • 工具调用需要授权弹窗：写文件、跑命令、推代码，弹窗在桌面上，你不在就卡住
  • 没有移动入口：开发商默认假设你 24 小时坐在电脑前

"凑合方案"为什么都不行

方案能让你远程发指令？能保留上下文？真实问题
SSH 到 Mac 跑 CLI能能手机打字痛苦，授权弹窗按不了
向日葵/腾讯会议远控能能盯小屏看大屏，眼瞎手痛
Claude 网页版 / Claw能✗ 每次新会话不知道你昨天在做啥
等开发商出移动端能（理论上）能等
真正的问题：不是"找一个能远程用的工具"，而是把"对话上下文"从一个 agent 的私有内存里抽出来——让任何 agent、任何设备都能接力。
1 · OVERVIEW

三件套：内存、入口、护栏

看似复杂的远程协作，本质只需三件东西：让 agent 的记忆能持久化、让你能从手机喊它、让它在自动跑的时候不闯祸。

🧠
PART ①

持久内存

把"对话记忆"从 LLM context 挪到磁盘文件。任何 agent、任何设备都能读同一份记忆，接力干活。

📱
PART ②

远程入口

让你从手机/任何设备触达正在跑的 agent。WorkBuddy 小程序是最低成本方案，其他场景可以自建桥。

🔒
PART ③

死命令护栏

放开高权限模式（自动通过授权）后，用 3 条以内的硬约束兜底——既能让 agent 跑得快，又不会闯祸。

三件套的关系

持久内存是地基（没它远程接力就是新会话）；远程入口是触达通道（没它你出门就联系不上）；死命令护栏是保险（没它远程模式不敢放开权限，等于半残）。三件缺一不可。

2.1 · MEMORY

持久内存：让 agent 不再失忆

所有 agent 都是失忆症患者——会话一关上下文清零。解法不是"找记忆力更强的 agent"，是把记忆写到磁盘文件。

两层记忆结构

.workbuddy/memory/
  ├── MEMORY.md           # 长期记忆：用户偏好、项目约定、死命令
  └── 2026-05-13.md       # 日记忆：今天做了啥、待办、对话决策记录
  └── 2026-05-12.md
  └── ...

MEMORY.md：跨会话不变的东西

  • 用户身份：你是谁、做什么的
  • 工作偏好：风格简洁/详尽？喜欢图表/纯文字？要不要拍马屁？
  • 项目约定：技术栈、命名规范、不能动的文件
  • 🔒 死命令（重要）：见 PART 3

YYYY-MM-DD.md：今天做了啥

每完成一件实质工作（改文件、部署、修 bug、做决策），立刻 append 一段。格式：

## HH:MM - 简短标题

做了什么（1-3 行）
为什么这么做（如果非显然）
踩了什么坑、得到了什么经验
关键洞察：这套文件结构不是"工具自带的功能"，而是你自己跟 agent 约定的协议。任何 agent（不管是 CodeBuddy 还是 Cursor 还是 Claude）只要被告知"读这里"，都能接得上。它把"记忆"从工具私域变成了通用资产。

双 agent 协作的核心：共享内存

你 Mac 上电脑前用 agent A，手机上远程入口接的是 agent B。两个 agent 是两个独立进程、不共享 context。但只要：

  1. A 做完事 append 到日记忆
  2. B 启动时第一件事读 MEMORY.md + 今天的日记忆

这两个 agent 就能无缝接力——A 改完一个 bug，B 在手机上接着收尾部署，毫无违和。这是远程协作的真正基础。

已知坑 1：小程序的"Claw"入口锁定一个工作目录（默认 ~/WorkBuddy/Claw）。如果你在多个项目目录跑 agent，小程序只会接到 Claw 目录里那个。

解法：让 Claw 目录里那个 agent 跨目录读其他项目的 memory，扮演"远程化身"——这是本文方法论的核心实践。
已知坑 2：小程序看不到 session 名（即使你 /rename 了），它只展示"任务"槽位。不要纠结 session 切换，让一个 agent 用持久内存接所有项目。
2.3 · CONSTRAINTS

死命令护栏：放开权限后的保险丝

远程协作有一个无法回避的矛盾：你在手机上点不了"允许"按钮。要么把权限放开（自动通过），要么 agent 卡死。

权限三档

模式文件编辑shell 命令建议场景
default弹窗确认弹窗确认你在电脑前
acceptEdits ⭐✓ 自动弹窗确认远程小改
bypassPermissions✓ 自动✓ 自动远程长任务，但必须配死命令

为什么必须配死命令

放到 bypassPermissions，agent 在手机指挥下可以"想到就做"，效率最高。但如果它判断错了一个命令（比如 rm -rf 误删），你来不及拦。

死命令 = 写在 MEMORY.md 顶部、永久优先级最高的硬约束。哪怕权限放开，只要触发死命令，agent 必须停手报告。

死命令公式（3 条够用）

第 1 条 · 保护本地文件：没用户明说，不删 .workbuddy/memory/ 以外的任何本地文件。包括 rm/rm -rf/mv 到 /tmp/git clean -fd 等同视为删除。

第 2 条 · 保护外部资源：不动云函数、不删数据库、不改 GitHub 仓库结构性设置（删/改分支保护/改可见性）。读取 + 内容更新可以，结构性删改一律 STOP。

第 3 条 · 透明跑命令：跑 bash 前一行话告诉用户要跑什么——哪怕 bypassPermissions 也要照做。这样你在手机看到命令能立刻喊停。

三条原则

  • 少而硬：3 条以内，超过没人记得住
  • 具体不空洞：写"不删 .workbuddy/memory 外文件"，不写"小心删除"
  • 放最顶部：MEMORY.md 第一个区块，每次开会话第一个读到
3 · CASE

案例：一个晚上的远程协作真实记录

2026-05-12 晚上，"二次编辑专项"方案稿从下午写到凌晨——前 6 小时在电脑前，最后 2 小时切换到远程模式。

时间线

时间事件哪个 agent
22:00合作方反馈"产品中心"概念太抽象，重写封面文案电脑前 agent
22:30资源表合作方看不懂，删"对接谁"列，加优先级标签电脑前 agent
23:00删掉"压力测试"小节（meta 内容不该塞进汇报稿）电脑前 agent
23:45手机视图表格列宽问题、SVG 太小要放大电脑前 agent
00:45启用远程模式：rename / bypassPermissions / 死命令写入 MEMORY切换中
01:00手机小程序发指令成功接管远程 agent
01:30本方法论站立项远程 agent

关键转折：发现"双 agent 抢入口"

启动远程模式后第一次发现 Mac 上跑着两个 CodeBuddy（一个在项目目录，一个在 Claw 目录），小程序只接到了后者——这迫使我们调整方案：不强迫合并 session，而是用共享内存让 Claw agent 远程"扮演"项目 agent。

方法论顿悟：不要试图"找到一个能控制所有 session 的入口"，那是工具问题。要让所有 agent 都成为同一个项目的化身——内存共享了，谁接你的指令都一样。

开场白模板（精简版 · 可复制）

身份：你是「二次编辑专项」远程化身。

启动动作：
1. 读 /Users/xxx/PROJECT/.workbuddy/memory/MEMORY.md（含死命令、用户偏好、项目约定）
2. 读今天 .workbuddy/memory/YYYY-MM-DD.md（接住最新进度）
3. 读完答："已接管 + 当前版本 + 上一件做的事"，一行话

工作规则：
- 我通过小程序找你都是「这个项目」的事
- 跑 bash 前先一行话告诉我要跑什么
- 不删 memory 外文件、不动云资源、不改 git 仓库设置（除非我明示）
- 完成每件实质工作立刻 append 到今天的日记忆
- 输出三行结论，不写长篇

开干。
4 · WHO

这套方法适合谁、不适合谁

不是所有人都需要远程指挥 agent。看清楚自己的工作模式再决定。

适合

  • 用 agent 做长链路项目：要跑好几天的事，期间会出差/通勤/睡觉
  • 白天会议多但晚上有灵感：躺床上想到改的地方，不想起来开电脑
  • 多设备人：电脑、平板、手机切换频繁
  • 不在工位时间长：高频出差、跨城工作

不适合

  • 用 agent 做单次小事：写完就完，根本用不上"远程接力"
  • 极度保密项目：不能把 PAT/SecretKey 放到能远程访问的环境（虽然本方法的远程通道是端到端加密，但保密性高的项目本来就不该自动化）
  • 不接受任何风险：再多死命令也不能 100% 防止 agent 失误，介意的人就别开 bypassPermissions

未来可扩展

  • 团队共用一个 agent：多人通过企微 Bot 给同一个 Mac 上的 agent 发指令
  • 跨工具迁移：今天 CodeBuddy，明天换 Cursor，memory 文件原样能用
  • 自动化结合：定时任务（如每天早上拉数据）+ 远程触发（你在地铁上喊它跑）
5 · TOOLKIT

配套工具：一键启用

把上面整套配置打包成一个 CodeBuddy skill，让你的同事一行命令上手。

Skill: claw-remote-takeover

安装位置：~/.workbuddy/skills/claw-remote-takeover/

用法：在任意 CodeBuddy 会话里发 /skill claw-remote-takeover，skill 自动注入开场白 + 引导你完成：

  • 检查/创建 .workbuddy/memory/MEMORY.md
  • 插入死命令模板（三条，可自定义）
  • 开 Gateway 给小程序
  • 切 bypassPermissions
  • 输出"远程化身就位"确认
