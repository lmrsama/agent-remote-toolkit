把 AI Agent
从电脑里 放出来

让你的 AI 助手不再绑在工位——出差、睡前、地铁、开会都能继续干活的三件套方法论。

By黎明睿（leomrli）
v1.02026-05-13
适用CodeBuddy / Cursor / Claude Code 用户
1 · WHY

痛点：AI Agent 绑在工位上

主流 AI 编程 agent（CodeBuddy / Cursor / Claude Code）都跑在桌面 IDE 里。三件事让你必须坐在电脑前用：

  • 上下文锁在内存里——会话一关 context 丢光，下次重讲一遍
  • 工具调用要授权弹窗——写文件 / 跑命令 / 推代码，弹窗在桌面，你不在就卡住
  • 没有移动入口——开发商默认你 24 小时坐工位
问题不是"找一个能远程用的工具"，是把对话上下文从 agent 的私有内存里抽出来——让任何 agent、任何设备都能接力。
2 · SOLUTION

解决方案：三件套

看似复杂的远程协作，本质三件东西。每件解决一个具体痛点。

PART ①

🧠 持久内存

把记忆从 LLM context 挪到磁盘文件

解决"会话一关 context 丢光"——任何 agent、任何设备读同一份记忆，无缝接力

PART ②

📱 远程入口

从手机触达正在跑的 agent

解决"没有移动入口"——WorkBuddy 小程序最低成本，一个二维码搞定

PART ③

🔒 死命令护栏

放开高权限模式后兜底

解决"授权弹窗按不了"——把权限放开自动通过，3 条死命令防止失控

三件如何咬合

缺一不可：没持久内存，远程接力是新会话；没远程入口，出门就联系不上；没死命令，远程模式不敢放权限等于半残。

3 · HOW

怎么用：4 步上手

给 CodeBuddy 用户的标准动作。其他 IDE 替换对应入口即可。

Step 1 · 装 skill

把仓库里的 skills/claw-remote-takeover/ 整个目录复制到本机：

cp -r skills/claw-remote-takeover ~/.workbuddy/skills/

Step 2 · 在当前项目里调用 skill

CodeBuddy 对话里发：

/skill claw-remote-takeover

它会自动：

  • 检查/创建 .workbuddy/memory/MEMORY.md
  • 把 3 条死命令模板写到顶部（已有的不重复）
  • 引导你开 Gateway + 切 bypassPermissions 权限

Step 3 · 手机绑小程序

对话里发 /gateway tunnel → 出二维码 → 手机微信扫码 → WorkBuddy 小程序自动绑这台 Mac。

Step 4 · 出门干活

关电脑 / 锁屏（不睡眠），手机微信 → WorkBuddy 小程序 → Claw 入口直接发指令，agent 接到就执行，文件交付推回小程序。

已知坑：小程序 Claw 入口锁定 ~/WorkBuddy/Claw 目录，如果你在多个项目里都跑 agent，小程序只接 Claw 那个。

解法：让 Claw 目录里的 agent 跨目录读其他项目的 memory，扮演"远程化身"——所以才需要持久内存这第一件套。
4 · CUSTOMIZE

自定义：要改什么、怎么改

默认配置 80% 场景能用。剩下 20% 按下面三个点调。

要改什么 改在哪 怎么改
3 条死命令 .workbuddy/memory/MEMORY.md 顶部 原版保护"本地文件 / 云资源 / bash 透明"。如果你项目有特殊禁区（比如不能动 prod 数据库、不能 push master），自己加一条。不要超过 5 条，超过 agent 记不住。
用户偏好 MEMORY.md 用户偏好区块 写下你的工作风格（详尽/简洁、要不要图表、是否需要拍马屁）。下次 agent 启动会读，行为就贴合你。
启动开场白 每次手机第一句话 默认模板见下方"开场白"。换项目时把"二次编辑专项"换成你自己的项目名 + memory 路径即可。

开场白模板

身份：你是「[项目名]」远程化身。

启动动作：
1. 读 [项目路径]/.workbuddy/memory/MEMORY.md
2. 读今天的 .workbuddy/memory/YYYY-MM-DD.md
3. 一行话答："已接管 + 当前版本 + 上一件做的事"

工作规则：
- 我从手机找你都是这个项目的事
- 跑 bash 前先一行话告诉我跑什么
- 完成每件实质工作立刻 append 到今天的日记忆
- 三行结论，不写长篇

开干。
