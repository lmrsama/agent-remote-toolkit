PART 1 · 背景

用户在哪一步卡住了？

先不讲解决方案——先看一个 AI agent 用户的完整工作日，看他在哪几个具体动作上撞了墙。

一个典型的 AI agent 用户的一天

完整 24 小时里，agent 只在「工位时间」真的能用——其余时段被两个痛点彻底锁死。

第 ③ 步和第 ⑤ 步是两堵墙。下面分别看是什么原因造成的：

痛点 1 · 出现在第 ③ 步

上下文锁在内存里

原因：agent 的上下文是 LLM 的私有内存，关闭/切换会话就丢；小程序又只有一个入口

WorkBuddy 小程序只有一个 Claw 入口。但 Mac 内存里你有 3 个项目、十几个会话——小程序无法精准定位你想接哪个，发出去的话进了哪个 agent 凭运气。

痛点 2 · 出现在第 ⑤ 步

工具调用要授权弹窗

原因：默认权限模式下，agent 写文件 / 跑命令 / 推 git 都会弹"允许 / 拒绝"窗，必须人在 Mac 屏幕前点

授权窗在 Mac 桌面上，你在外面看不到也按不了。agent 就在那里干转圈、所有任务停摆，直到你回家手动点一下"允许"。

这才是真正的问题：不是"找一个能远程用的工具"——而是怎么让 agent 记得住、跑得动。
PART 2 · 解决方案

三件套：每件一句话

两个解法解两个痛点，加一条工作纪律保接力。

01MEMORY
✓ 解痛点 1

记忆上磁盘

把对话上下文从 LLM 内存抽出来，写到 .workbuddy/memory/ 下的 markdown 文件——任何 agent 读同一份。

⚡ Highlight：这是协议而非工具功能，CodeBuddy / Cursor / Claude Code 通用。

02CONSTRAINTS
✓ 解痛点 2

三条死命令

把权限放到 bypassPermissions 自动通过授权，3 条硬约束兜底防失控。

死命令 ①

不删本地文件

仅 .workbuddy/memory/ 内可改，其他文件用户不明说不删。

死命令 ②

不动外部资源

不删云函数 / 数据库 / 仓库结构。读 + 内容更新可以。

死命令 ③

跑 bash 前先报告

跑命令前一行话告诉用户跑什么——手机能随时喊停。

⚡ Highlight：少而硬（3 条够用）、写在 MEMORY.md 顶部、bypassPermissions 也不豁免。

03DISCIPLINE
✓ 让 ① + ② 真正闭环

立即覆写 memory

每完成一件实质工作（改文件 / 部署 / 修 bug），立即 append 到当天日记忆——不批量、不延后。

⚡ Highlight：跨端接力的命脉。电脑写晚一秒，手机读到的就是空气。

PART 3 · 用户上手

0 门槛四步走

深色字 = 你要做的；浅灰字 = 背后发生了啥（不用理解，只是好奇可以看）。

第 0 步 · 装 skill

终端粘一行：

curl -fsSL https://raw.githubusercontent.com/lmrsama/agent-remote-toolkit/main/install.sh | bash

脚本下载 SKILL.md 到 ~/.workbuddy/skills/claw-remote-takeover/。已装会备份旧版。

第 1 步 · 在项目里发指令

CodeBuddy 对话框发：

/skill claw-remote-takeover

跟着它的提示走完两件事就行：① 微信搜「WorkBuddy」绑定本机；② 在输入框发 /config set permissions.mode bypassPermissions。完成后告诉它"好了"。

背后发生：skill 帮你建 .workbuddy/memory/MEMORY.md，往顶部写 3 条死命令 + 1 条覆写 memory 工作纪律，然后等你完成手机绑定 + 放权限两步。

第 2 步 · 复制开场白

skill 会把开场白文本现成生成给你（项目名、路径都填好了），原样复制就行。

背后发生：开场白里包含读哪个 memory 路径、要遵守哪些规则——任意 agent 看到都能扮演这个项目的远程化身。

第 3 步 · 手机粘贴

微信 → WorkBuddy 小程序 → Claw 入口 → 粘贴开场白发出去。地铁上、床上、咖啡厅，随时给指令，agent 接到就干。

背后发生：手机的 agent 读 MEMORY + 今天日记忆 → 一行话回你"已接管"。它干完写日记忆，电脑前打开依然能接。

PART 4 · Skill 部署

怎么装：三档方式按门槛挑

所有动作 skill 已经封好了。装一次终身可用。下面三种方式任选其一。

方式 A · 一键安装（推荐）

打开终端，粘贴下面这一行回车即可。脚本会自动下载 skill 到 ~/.workbuddy/skills/claw-remote-takeover/：

curl -fsSL https://raw.githubusercontent.com/lmrsama/agent-remote-toolkit/main/install.sh | bash

已装过会自动备份旧版（claw-remote-takeover.bak.时间戳），不会丢。

方式 B · 手动下载（不会用终端的同事用）

  1. 下载 SKILL.md：点这里 （右键 → "链接另存为"）
  2. 找到本机 ~/.workbuddy/skills/ 目录（macOS：Finder 按 ⌘+Shift+G 输入路径；Windows：资源管理器地址栏粘贴 %USERPROFILE%\.workbuddy\skills）
  3. 在里面新建文件夹 claw-remote-takeover
  4. 把刚才下载的 SKILL.md 拖进去

方式 C · git clone（工程师习惯）

git clone https://github.com/lmrsama/agent-remote-toolkit.git
cp -r agent-remote-toolkit/skills/claw-remote-takeover ~/.workbuddy/skills/

装完之后

  1. 重启一次 CodeBuddy（让它扫到新 skill）
  2. 进入想远程化的项目目录
  3. 对话框发 /skill claw-remote-takeover
  4. 跟着 skill 引导走完三步（写 MEMORY、开 gateway、放权限），它会返回开场白文本给你
  5. 开场白复制粘贴到手机微信 → WorkBuddy 小程序 → Claw 入口 → 远程指挥就绪

需要自定义的点

改什么改在哪怎么改
3 条死命令 .workbuddy/memory/MEMORY.md 顶部 原版保护"本地文件 / 云资源 / bash 透明"。项目有特殊禁区（不能动 prod DB、不能 push master）自己加一条。不超过 5 条。
用户偏好 MEMORY.md 用户偏好区块 写下你的工作风格（详尽/简洁、要图表、要不要拍马屁）。下次 agent 启动会读，行为会贴合你。
开场白 每次手机第一句话 skill 跑完自动给你。换项目时把项目名、memory 路径换掉即可。
