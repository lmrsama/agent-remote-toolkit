把 AI Agent
从电脑里 放出来

让你的 AI 助手不再绑在工位——出差、睡前、地铁、开会都能继续干活的三件套方法论。

By黎明睿（leomrli）
v1.22026-05-13
适用CodeBuddy / Cursor / Claude Code 用户
1 · WHY

两个共同痛点，把你绑在工位上

不论用 CodeBuddy、Cursor 还是 Claude Code，远程指挥 AI agent 时都会撞上同样两堵墙。

痛点 1

上下文锁在内存里

WorkBuddy 小程序只有一个 Claw 入口，但你 Mac 上一会话开十几个 agent。小程序不知道该接哪个——你昨天和它聊到哪、改到哪，它一概不清楚。

痛点 2

工具调用要授权弹窗

agent 想写文件、跑 shell、推 git——任何"动作"都会弹授权窗，必须在 Mac 屏幕上点。你在外面看不到也按不了，agent 就在那里干转圈，所有任务停摆。

真正的问题不是"找一个能远程用的工具"，而是怎么让 agent 记得住、跑得动——把记忆抽出来、把授权放开但兜住。
2 · SOLUTION

三件套：磁盘内存 + 小程序入口 + 三条死命令

前两件解痛点 1（接力），后一件解痛点 2（授权 + 兜底）。三件咬合，缺一不可。

01MEMORY

记忆上磁盘

把对话上下文从 LLM 的私有内存抽出来，写到磁盘上的 markdown 文件。任何 agent、任何会话、任何设备都能读同一份。

✓ 解决痛点 1：跨端跨会话也能精准接上

关键认知：这不是某个工具自带的功能，是你和 agent 约定的协议——任何 agent 只要被告知"读这里"，都能接上。把记忆从工具私域变成通用资产。

02CHANNEL

WorkBuddy 小程序

CodeBuddy 自带的远程入口——发 /gateway tunnel 出二维码，手机微信扫码即用。

✓ 与 ① 配合：手机喊一声，agent 读 memory 就接上

人人皆知，不展开。关键是和"磁盘内存"配合：让 Claw 入口接到的那个 agent 主动读你项目里的 MEMORY，扮演项目的"远程化身"。

03CONSTRAINTS

三条死命令

把权限放到 bypassPermissions（所有授权自动通过），用 3 条硬约束兜底。

✓ 解决痛点 2：手机点不了授权 → 直接放开 + 三条防失控
死命令 ①

不删本地文件

不删 .workbuddy/memory/ 以外的任何本地文件，除非用户明说。
解决：防 rm 误删项目文件。

死命令 ②

不动外部资源

不删云函数 / 数据库、不动 GitHub 仓库结构性设置。读 + 内容更新可以，结构性删改一律 STOP。
解决：防 agent 误删 prod 资源。

死命令 ③

跑 bash 前先报告

跑任何 bash 命令前，先一行话告诉用户要跑什么。
解决：权限放开后用户仍能在手机上看到命令、随时喊停。

原则：少而硬（3 条够用，超过没人记得住）、写在 MEMORY.md 顶部（每次开会话第一个读到）、bypassPermissions 也不豁免。

3 · HOW

三步上手：从电脑到手机的接力

第 1、2 步在电脑前做完一次（每个新项目一次）；第 3 步以后每次出门复制粘贴即可。

第 1 步 · 在 PC 项目里跑 skill

CodeBuddy 对话里发：

/skill claw-remote-takeover

skill 自动做三件事：

  • 检查/创建 .workbuddy/memory/MEMORY.md
  • 把 3 条死命令写到顶部
  • 引导你开 /gateway tunnel + 切 bypassPermissions

跑完 skill 会直接返回开场白文本给你——不用自己编辑，复制就能用。

第 2 步 · 复制 skill 返回的开场白

形如下面这段（项目名、路径已自动填好）：

身份：你是「[项目名]」远程化身。
启动：读 [路径]/.workbuddy/memory/MEMORY.md + 今天的日记忆，
       一行话答："已接管 + 当前版本 + 上一件做的事"。
规则：跑 bash 前先报告 / 完成立刻 append 日记忆 / 三行结论。
开干。

第 3 步 · 手机 WorkBuddy 小程序粘贴开场白

地铁、床上、咖啡厅——掏出手机：

  • 微信 → WorkBuddy 小程序 → Claw 入口
  • 粘贴上一步的开场白发出去
  • agent 读完 memory 一行话回你"已接管 + 当前状态"
  • 你直接发任务，agent 干完写日记忆，下次电脑前你打开依然能接上
核心闭环：电脑写 → 磁盘存 → 手机读 → 手机写 → 磁盘存 → 电脑读。
两个 agent，一份内存，无缝接力。
4 · DEPLOY

Skill 怎么装、要改什么

所有标准动作 skill 已经封好了。装一次终身可用。

安装

# 克隆仓库
git clone https://github.com/lmrsama/agent-remote-toolkit.git

# 复制 skill 到本机
cp -r agent-remote-toolkit/skills/claw-remote-takeover ~/.workbuddy/skills/

调用

在任意 CodeBuddy 会话里发 /skill claw-remote-takeover。每个新项目调用一次（写一次 MEMORY 就行了）。

需要自定义的点

改什么改在哪怎么改
3 条死命令 .workbuddy/memory/MEMORY.md 顶部 原版保护"本地文件 / 云资源 / bash 透明"。项目有特殊禁区（不能动 prod DB、不能 push master）自己加一条。不超过 5 条。
用户偏好 MEMORY.md 用户偏好区块 写下你的工作风格（详尽/简洁、要图表、要不要拍马屁）。下次 agent 启动会读，行为会贴合你。
开场白 每次手机第一句话 skill 跑完自动给你。如换项目，把项目名、memory 路径换掉即可。
