PART 1 · 背景

用户在哪一步卡住了？

先不讲解决方案——先看一个 AI agent 用户的完整工作日，看他在哪几个具体动作上撞了墙。

一个典型的 AI agent 用户的一天

完整 24 小时里，agent 只在「工位时间」真的能用——其余时段被两个痛点彻底锁死。

第 ③ 步和第 ⑤ 步是两堵墙。下面分别看是什么原因造成的：

痛点 1 · 出现在第 ③ 步

上下文锁在内存里

原因：agent 的上下文是 LLM 的私有内存，关闭/切换会话就丢；小程序又只有一个入口

WorkBuddy 小程序只有一个 Claw 入口。但 Mac 内存里你有 3 个项目、十几个会话——小程序无法精准定位你想接哪个，发出去的话进了哪个 agent 凭运气。

痛点 2 · 出现在第 ⑤ 步

工具调用要授权弹窗

原因：默认权限模式下，agent 写文件 / 跑命令 / 推 git 都会弹"允许 / 拒绝"窗，必须人在 Mac 屏幕前点

授权窗在 Mac 桌面上，你在外面看不到也按不了。agent 就在那里干转圈、所有任务停摆，直到你回家手动点一下"允许"。

这才是真正的问题：不是"找一个能远程用的工具"——而是怎么让 agent 记得住、跑得动。
PART 2 · 解决方案

三件套：每一件对应一个痛点

不要被"三件套"这个数字唬住——其实是两个解法解两个痛点，加上第三个兜底护栏。

01MEMORY
✓ 解痛点 1：上下文锁内存

记忆上磁盘

把对话上下文从 LLM 的私有内存抽出来，写到磁盘上的 markdown 文件——任何 agent、任何会话、任何设备读同一份。

为什么解了痛点 1？——小程序定位不准没关系，它接到哪个 agent 都能读同一份 memory。原本"找不到那个 agent"的问题，变成"任何 agent 都是那个 agent"。

关键认知：这不是某个工具自带的功能，是你和 agent 约定的协议——任何 agent 只要被告知"读这里"，都能接上。把记忆从工具私域变成通用资产。

02CONSTRAINTS
✓ 解痛点 2：授权弹窗按不了

三条死命令

把权限放到 bypassPermissions（所有授权自动通过），用 3 条硬约束兜底——既能让 agent 自己干完，又不会闯祸。

为什么解了痛点 2？——不是把弹窗"做好按"，而是直接放开自动授权。但放开就有失控风险，所以用三条死命令兜底。

死命令 ①

不删本地文件

不删 .workbuddy/memory/ 以外的任何本地文件（除非用户明说）。

防：rm -rf 误删项目文件
死命令 ②

不动外部资源

不删云函数 / 数据库、不动 GitHub 仓库结构性设置。读 + 内容更新可以。

防：误删 prod 资源 / 误改仓库可见性
死命令 ③

跑 bash 前先报告

跑任何 bash 命令前，先一行话告诉用户要跑什么。

防：权限放开后用户失去监督——靠这条把命令"广播"出去，用户在手机上能随时喊停

原则：少而硬（3 条够用，超过没人记得住）、写在 MEMORY.md 顶部（每次开会话第一个读到）、bypassPermissions 也不豁免。

03HANDOFF
✓ 把 ① + ② 串起来：用户体验闭环

开场白接力

让用户在 PC 跑完 skill 后自动拿到一段开场白文本，复制粘贴到手机即可让任意 agent 扮演项目化身。

① 提供了"共享内存"，② 提供了"自动跑权限"，但中间还差一步：用户怎么告诉手机那边的 agent 去读这份内存？答案就是这段开场白——一段标准化提示，让任意 agent 看到就知道"我应该读哪个项目的 memory，按什么规则干活"。

PART 3 · 用户上手

三步：从电脑配好到手机接力

第 1、2 步在电脑前每个新项目做一次；第 3 步以后每次出门复制粘贴即可。

第 1 步 · 在 PC 项目里跑 skill

CodeBuddy 对话里发：

/skill claw-remote-takeover

skill 自动做这些：

  • 检查/创建 .workbuddy/memory/MEMORY.md
  • 把 3 条死命令写到顶部
  • 提示你去微信搜「WorkBuddy」小程序、扫码绑定本会话
  • 引导你在输入框发 /gateway tunnel 出二维码、发 /config set permissions.mode bypassPermissions 放权限
为什么是引导而非自动？/gateway 和 /config 是 CodeBuddy CLI 的内置斜杠命令，必须由你在输入框里手动发——agent 没有这两个命令的执行权限。

第 2 步 · 复制 skill 返回的开场白

当你完成 step 1 的三件事告诉 skill「好了」，它会把开场白文本现成生成好返还给你，形如：

身份：你是「[项目名]」远程化身。

启动：读 [路径]/.workbuddy/memory/MEMORY.md + 今天的日记忆，
      一行话答："已接管 + 当前版本 + 上一件做的事"。

规则：跑 bash 前先报告 / 完成立刻 append 日记忆 / 三行结论。

开干。

项目名、路径已经按你 PC 当前的工作目录自动填好——不用自己编辑，复制就能用。

第 3 步 · 手机粘贴到 WorkBuddy 小程序

地铁、床上、咖啡厅——掏出手机：

  • 微信 → WorkBuddy 小程序 → Claw 入口
  • 粘贴上一步的开场白发出去
  • agent 读完 memory，一行话回你"已接管 + 当前状态"
  • 你直接发任务，agent 干完写日记忆，下次电脑前你打开依然能接上
核心闭环：电脑写 → 磁盘存 → 手机读 → 手机写 → 磁盘存 → 电脑读。
两个 agent，一份内存，无缝接力。
PART 4 · Skill 部署

怎么装、要改什么

所有动作 skill 已经封好了。装一次终身可用。

安装

# 克隆仓库
git clone https://github.com/lmrsama/agent-remote-toolkit.git

# 复制 skill 到本机
cp -r agent-remote-toolkit/skills/claw-remote-takeover ~/.workbuddy/skills/

调用

在任意 CodeBuddy 会话里发 /skill claw-remote-takeover。每个新项目调用一次。

需要自定义的点

改什么改在哪怎么改
3 条死命令 .workbuddy/memory/MEMORY.md 顶部 原版保护"本地文件 / 云资源 / bash 透明"。项目有特殊禁区（不能动 prod DB、不能 push master）自己加一条。不超过 5 条。
用户偏好 MEMORY.md 用户偏好区块 写下你的工作风格（详尽/简洁、要图表、要不要拍马屁）。下次 agent 启动会读，行为会贴合你。
开场白 每次手机第一句话 skill 跑完自动给你。换项目时把项目名、memory 路径换掉即可。
